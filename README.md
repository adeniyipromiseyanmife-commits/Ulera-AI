# What is Ùlẹ̀rà AI?

Ùlẹ̀rà AI is a mobile-first intelligent health platform designed to address two of Nigeria’s most pressing healthcare challenges:

- **87% health insurance coverage gap**
- **69% healthcare worker burnout rate**

Built by a licensed Community Health Extension Worker (CHEW) with five years of frontline experience in Oyo State, Ùlẹ̀rà AI adapts health guidance, burnout support, and insurance awareness to a user's profession, location, and life stage.

---

# The Problem

## The Insurance Gap
- Only **13% of Nigerians** have health insurance.
- More than **190 million Nigerians** remain uninsured.
- **79% of adults** pay healthcare expenses out-of-pocket.
- Existing schemes such as **BHCPF** and state health insurance programmes remain underutilised due to low awareness.

## The Burnout Crisis
- **69% of healthcare workers** experience burnout.
- Nigeria has only **2.9 doctors per 10,000 people**, far below WHO recommendations.
- Healthcare professionals often work without adequate digital support systems.

# NEW: Burnout Intelligence System

Ùlẹ̀rà AI now includes a structured burnout assessment and monitoring engine for healthcare professionals.

## Components

| File | Purpose |
|--------|---------|
| burnout-assessment.html | Weekly burnout check-in |
| burnout-engine.js | PHQ-9 + PSS-10 + GAD-7 scoring |
| burnout-storage.js | Local data storage and weekly tracking |
| burnout-gauge.js | Dashboard burnout gauge |
| burnout.css | Assessment and gauge styling |

## Burnout Assessment Flow

1. User opens dashboard
2. Burnout Gauge displays latest score
3. User completes weekly assessment
4. PHQ-9, PSS-10 and GAD-7 questionnaires are processed
5. Burnout Risk Index (BRI) is generated
6. Results are visualised using an animated gauge
7. Data is stored locally for trend analysis
8. Weekly follow-up reminders are triggered

## Dashboard Burnout Gauge

Provides:

- Burnout Risk Score
- Risk Classification
- Personalised Support Messages
- Trend Monitoring
- Weekly Check-in Prompts

## Privacy & Safety

- All assessment data remains in browser local storage
- No personal assessment data is transmitted to servers
- Safety alerts appear automatically when PHQ-9 Item 9 indicates risk concerns
- Users can clear all stored data at any time
